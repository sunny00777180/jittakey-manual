JittaKey 使用説明書 (日本語)

✨ 概要: WYSIWYG 日本語入力の革新

JittaKeyは、従来のローマ字変換方式を根本から見直した、直感的(WYSIWYG)日本語入力方式を提供します。

  • 入力したとおりに表示 (What You See Is What You Get)
  • ローマ字ベースのユーザーにもすぐに適応可能
  • 入力回数を約20%以上削減
  • 視覚的な連想法で学習もスムーズ

🔧 JittaKey Input Principles

  • Principle 1: WYSIWYG, meaning that every character displayed on the keyboard is input with one keystroke.
    Examples:
    あいうえお ぁぃぅぇぉ
    わやゆょ ゎゃゅょ
    んをｰっヴ
    きぎしじちぢにひびぴみり
    ゕゖ「」~^_^！、。？
  • Principle 2: Romanized input is applied when ‘い consonants -きぎしじちぢにひびぴみり’ + vowels (あうえお) are entered in sequence.
    Examples: き + あ = か, き + う= く, き + え = け, き + お = こ
    り + あ = ら, り + う= る, り + え = れ, り + お = ろ
  • Principle 3: When entering ‘い consonants’ + vowels (あいうえお), the consonant is decided as い or (sp), and then the vowels (あ,い,う,え,お) are entered.
    Examples: To enter きあ, type きいあ or き(sp)あ. To type きい, enter きいい or き(sp)い.
  • Principle 4: Yōon (small vowel sounds) are independently input. This is completely different from combining yōon in Romaji input.
    Examples: きょう=き+ょ+う is entered as shown.
    Based on these principles, JittaKey offers a 20% efficiency improvement over Romaji.
    For example, to type “おききします。” in Romaji, it’s entered as okikisimasu + change keymap to input “。”.
    However, with JittaKey, it’s entered as okksmasu。without changing the keymap.
    In terms of 'keystrokes', it is processed in 9 keystrokes, compared to 11 keystrokes + keymap change + 1 keystroke in Romaji.
  • Principle 5: JittaKey performs exceptionally well on mobile messengers like Line, Whatsapp, Kakaotalk, etc.
    Special characters “「 」 ~ ^_^ っ ！ 、 。 ？” can be input with a 0.2-second long press or shift+select method, without changing the keymap.
    Example: 「今日はいい天気で、よかった」んですね~~^_^！
    This allows the sentence above to be entered without changing the keymap.
    Not only does it improve speed, but it also reduces the ‘stress’ of having to switch keymaps every time special characters are needed, offering a more comfortable ‘messenger experience’.
  • Others: Full-width and half-width Katakana, numbers, and English are perfectly implemented with WYSIWYG, so everything is input exactly as displayed.
